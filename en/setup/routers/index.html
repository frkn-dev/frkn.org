VPN on router

Connect your entire home internet through one router. Step-by-step instructions for Xray and AmneziaWG on OpenWRT and Keenetic.

What you'll need

  • Router with OpenWRT 23.05+ or 24.10+
  • SSH access (root login and password)
  • LuCI web panel
  • ~30 MB free space on the router
  • AmneziaWG config from your FRKN personal account

1 Connect via SSH

On your computer in the terminal run:

ssh root@192.168.1.1
If SSH asks about host confirmation — enter yes. The password is the one you set during OpenWRT setup.

2 Install AmneziaWG

Run the install script:

sh <(wget -O - https://raw.githubusercontent.com/Slava-Shchipunov/awg-openwrt/refs/heads/master/amneziawg-install.sh)
During the script it will ask questions:
• Russian language pack — choose as desired
• Set up the AWG interface now — answer n, we'll configure the interface via LuCI

3 Reboot the router

reboot

Wait 30–60 seconds for the router to boot up.

4 Create interface in LuCI

Open the router web panel → Network → Interfaces → Add new interface.

  • Name: awg0 (or any other)
  • Protocol: AmneziaWG VPN
  • Click Create interface
If the AmneziaWG protocol is not in the list — clear the browser cache or re-login to LuCI.

5 Import configuration

In the interface settings click Load configuration and upload the .conf file from your personal account, or paste its contents.

Make sure that AmneziaWG protocol is selected in your personal account when downloading the config.

6 Configure peer and routing

  • Advanced Settings tab → enable Use default gateway
  • Peers tab → Edit → enable Route Allowed IPs
  • Click Save → Save & Apply

7 Configure firewall

Go to Network → Firewall → Zones → Add:

  • Name: VPN
  • Input: reject
  • Output: accept
  • Forward: reject
  • Enable IPv4 Masquerading and MSS clamping
  • Covered networks: select the created interface (awg0)
  • Allow forward from source zones: select lan
Important: the field Allow forward from source zones, not to destination zones. These are two different fields located one below the other.

8 Apply and check

Click Save & Apply, then reboot the router via System → Reboot.

After booting, check the IP on any device — it should match the server location.

What you'll need

  • Router with OpenWRT 23.05+ or 24.10+
  • At least 128 MB RAM (256 MB recommended)
  • ~30 MB free space
  • SSH access
  • Xray link or config from your FRKN personal account

1 Connect via SSH

ssh root@192.168.1.1

2 Add v2rayA repository

wget https://downloads.sourceforge.net/project/v2raya/openwrt/v2raya.pub -O /etc/opkg/keys/94cc2a834fb0aa03
echo "src/gz v2raya https://downloads.sourceforge.net/project/v2raya/openwrt/$(. /etc/openwrt_release && echo "$DISTRIB_ARCH")" | tee -a "/etc/opkg/customfeeds.conf"

3 Update package list

opkg update

4 Install v2rayA and Xray

opkg install v2raya xray-core kmod-nft-tproxy iptables-nft
If you want to manage v2rayA directly from LuCI, additionally install:
opkg install luci-app-v2raya

5 Start v2rayA

uci set v2raya.config.enabled='1'
uci commit v2raya
/etc/init.d/v2raya enable
/etc/init.d/v2raya start

6 Open web interface

Go to: 

http://192.168.1.1:2017

On first login create an administrator login and password.

If the page says loading geoip.dat / geosite.dat — wait a minute and refresh.

7 Import connection

In your FRKN personal account select the Xray protocol, copy the link or download the config.

In v2rayA go to Subscriptions → Add, paste the link and update. Or import the config manually via Import.

8 Enable proxy

Select the desired server in the list → click Not running at the top of the page to switch to Running.

Transparent proxy mode (Transparent Proxy / System Proxy) is usually enabled by default — all LAN traffic will go through Xray.

9 Check operation

On any device on the network check the IP address — it should match the server location.

If something doesn't work, check the logs:
logread -f | grep v2raya

What you'll need

  • KeeneticOS 5.1 or newer
  • Installed WireGuard VPN component
  • AmneziaWG config from your FRKN personal account

1 Install WireGuard component

Management → System settings → Components → find WireGuard VPN → install.

After installation the router will reboot.

2 Download AmneziaWG config

In your FRKN personal account select the AmneziaWG protocol and download the .conf file.

3 Import connection

Internet → Other connections → WireGuard → Import from file. Select the downloaded .conf file.

4 Configure internet usage

Tap the created connection → enable Use for internet access → Save.

Do not enable the connection itself yet.

5 Create connection policy

Go to Internet → Connection policies → Add policy. Name it, for example, VPN.

  • Select the created AWG connection and drag it to the top of the list
  • If you need a fallback — leave ISP checked; if killswitch — uncheck

6 Bind devices

In the same policy go to the Policy binding tab and drag the needed devices into the VPN policy.

7 Enable connection

Internet → Other connections → WireGuard → enable the created connection. After a few seconds the status should turn green.

What you'll need

  • KeeneticOS 4.2 or newer
  • Installed WireGuard VPN component
  • AmneziaWG config from your FRKN personal account

1 Create WireGuard connection

Follow steps 1–4 of the automatic variant: install the component, import the .conf, enable "Use for internet access", but do not enable the connection itself.

2 Find interface name

Open the router web CLI (gear → Command line) and run: 

show interface

Find your connection in the output by name and remember the interface-name value (e.g., Wireguard0).

3 Apply AmneziaWG parameters

From the config take the values Jc, Jmin, Jmax, S1, S2, H1–H4.

interface Wireguard0 wireguard asc 5 42 54 0 0 1 2 3 4
Replace Wireguard0 and the numbers with your values from the configuration file.

4 Save and activate

system configuration save

Then configure the connection policy (steps 5–7 of the automatic variant) and enable the connection.

What you'll need

  • Keenetic with a USB port
  • Enabled Entware component
  • AmneziaWG config from your FRKN personal account

1 Install Entware

Management → System settings → Components → install Entware.

2 Install AWG Manager

Connect via SSH to the router and run:

opkg update && opkg upgrade
opkg install curl
curl -sL https://raw.githubusercontent.com/hoaxisr/awg-manager/master/scripts/install.sh | sh
If GitHub is unavailable, use the mirror:
wget -qO- http://repo.hoaxisr.ru/install.sh | sh

3 Open web interface

After installation AWG Manager is available at the router address on port 2222. 

http://192.168.1.1:2222

4 Import config

In the AWG Manager web interface add a new tunnel and import the .conf file from your FRKN personal account.

What you'll need

  • Keenetic with a USB port
  • USB drive with EXT4 filesystem (preferably ext4)
  • Enabled OPKG / Entware component
  • SSH access to the router
  • Xray link or config from your FRKN personal account

1 Install Entware

Prepare a USB flash drive in EXT4 format, connect it to the router.

Management → System settings → Components → enable OPKG and Entware.

In the router web interface go to the OPKG page, select the drive and specify initrc: /opt/etc/init.d/rc.unslung.

2 Connect via SSH

Login: root, default password: keenetic (change immediately via passwd). 

ssh root@192.168.1.1
Port 222 if the SSH Server component is installed. If not — port 22.

3 Install XKeen

opkg update && opkg upgrade && opkg install curl tar && cd /tmp
sh -c "$(curl -sSL https://raw.githubusercontent.com/jameszeroX/XKeen/main/install.sh)"
If GitHub is unavailable, use the alternative method:
curl -OL https://github.com/jameszeroX/XKeen/releases/latest/download/xkeen.tar.gz && tar -xvzf xkeen.tar.gz -C /opt/sbin && rm xkeen.tar.gz && xkeen -i

4 Choose GeoIP / GeoSite

During XKeen installation it will ask about GeoIP and GeoSite — it is recommended to install both (choose option 1).

Then enable automatic update (option 1) and specify a convenient time.

5 Get Xray config

In your FRKN personal account select the Xray protocol and copy the subscription link or download the JSON config.

XKeen uses files in the directory /opt/etc/xray/configs/. The main outbound config is usually named 04_outbounds.json.

6 Upload config to router

Via SCP / WinSCP or SFTP upload the ready config to: 

/opt/etc/xray/configs/config.json

Or, if you have a link like vless://..., use a config generator (e.g., RouteSense XKeen config generator) and put the resulting JSON in the specified folder.

7 Start XKeen

xkeen -start

Or restart if you changed the config:

xkeen -restart

Check status:

xkeen -status

8 Configure connection policy

Go to the router web interface:

  • Connection priorities → Internet access policies → create a policy, for example, XKeen
  • In the policy select the internet access method — check the provider
  • Policy application → add devices or the entire network to the XKeen policy
XKeen works as a local proxy on the router, so devices must be directed to it via an access policy.

9 Move Keenetic services from port 443

If you use XTLS-Reality, Keenetic may occupy port 443. Move it, for example, to 8443: 

ip http ssl port 8443
system configuration save
After this KeenDNS and other services will be available at xxxx.keenetic.link:8443.

10 Check operation

On a device in the XKeen policy check the IP address — it should match the server location.

XKeen logs can be viewed with:
xkeen -diag or logread | grep xray
